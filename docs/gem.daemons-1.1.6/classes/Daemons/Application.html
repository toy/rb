exception_log()

This is a nice little function for debugging purposes: In case a multi-threaded ruby script exits due to an uncaught exception it may be difficult to find out where the exception came from because one cannot catch exceptions that are thrown in threads other than the main thread.

This function searches for all exceptions in memory and outputs them to STDERR (if it is connected) and to a log file in the pid-file directory.

running?()

This function implements a (probably too simle) method to detect whether the program with the pid found in the pid-file is still running. It just searches for the pid in the output of ps ax, which is probably not a good idea in some cases. Alternatives would be to use a direct access method the unix process control system.

     # File lib/daemons/application.rb, line 377
377:     def stop(no_wait = false)
378:       if not running?
379:         self.zap
380:         return
381:       end
382:       
383:       pid = @pid.pid
384:       
385:       # Catch errors when trying to kill a process that doesn't
386:       # exist. This happens when the process quits and hasn't been
387:       # restarted by the monitor yet. By catching the error, we allow the
388:       # pid file clean-up to occur.
389:       begin
390:         Process.kill(SIGNAL, pid)
391:       rescue Errno::ESRCH => e
392:         puts "#{e} #{pid}"
393:         puts "deleting pid-file."
394:       end
395:       
396:       if not no_wait
397:         if @force_kill_waittime > 0
398:           puts "#{self.group.app_name}: trying to stop process with pid #{pid}..."
399:           STDOUT.flush
400:           
401:           begin
402:             Timeout::timeout(@force_kill_waittime) {
403:               while Pid.running?(pid)
404:                 sleep(0.2)
405:               end
406:             }
407:           rescue Timeout::Error
408:             puts "#{self.group.app_name}: process with pid #{pid} won't stop, we forcefully kill it..."
409:             STDOUT.flush
410:             
411:             begin
412:               Process.kill('KILL', pid)
413:             rescue Errno::ESRCH
414:             end
415:             
416:             begin
417:               Timeout::timeout(20) {
418:                 while Pid.running?(pid)
419:                   sleep(1)
420:                 end
421:               }
422:             rescue Timeout::Error
423:               puts "#{self.group.app_name}: unable to forcefully kill process with pid #{pid}."
424:               STDOUT.flush
425:             end
426:           end
427:         end
428:         
429:         
430:       end
431:       
432:       sleep(0.1)
433:       unless Pid.running?(pid)
434:         # We try to remove the pid-files by ourselves, in case the application
435:         # didn't clean it up.
436:         begin; @pid.cleanup; rescue ::Exception; end
437:         
438:         puts "#{self.group.app_name}: process with pid #{pid} successfully stopped."
439:         STDOUT.flush
440:       end
441:       
442:     end

Class Vagrant::Environment < Object

  • lib/vagrant/environment.rb

Represents a single Vagrant environment. A “Vagrant environment” is defined as basically a folder with a “Vagrantfile.” This class allows access to the VMs, CLI, etc. all in the scope of this environment.

load_config!()

Loads this environment’s configuration and stores it in the {config} variable. The configuration loaded by this method is specified to this environment, meaning that it will use the given root directory to load the Vagrantfile into that context.

Source: show 

     # File lib/vagrant/environment.rb, line 335
335:     def load_config!
336:       # Initialize the config loader
337:       config_loader = Config::Loader.new
338:       config_loader.load_order = [:default, :box, :home, :root, :vm]
339: 
340:       inner_load = lambda do |*args|
341:         # This is for Ruby 1.8.7 compatibility. Ruby 1.8.7 doesn't allow
342:         # default arguments for lambdas, so we get around by doing a *args
343:         # and setting the args here.
344:         subvm = args[0]
345:         box   = args[1]
346: 
347:         # Default Vagrantfile first. This is the Vagrantfile that ships
348:         # with Vagrant.
349:         config_loader.set(:default, File.expand_path("config/default.rb", Vagrant.source_root))
350: 
351:         if box
352:           # We load the box Vagrantfile
353:           box_vagrantfile = find_vagrantfile(box.directory)
354:           config_loader.set(:box, box_vagrantfile) if box_vagrantfile
355:         end
356: 
357:         if home_path
358:           # Load the home Vagrantfile
359:           home_vagrantfile = find_vagrantfile(home_path)
360:           config_loader.set(:home, home_vagrantfile) if home_vagrantfile
361:         end
362: 
363:         if root_path
364:           # Load the Vagrantfile in this directory
365:           root_vagrantfile = find_vagrantfile(root_path)
366:           config_loader.set(:root, root_vagrantfile) if root_vagrantfile
367:         end
368: 
369:         if subvm
370:           # We have subvm configuration, so set that up as well.
371:           config_loader.set(:vm, subvm.proc_stack)
372:         end
373: 
374:         # Execute the configuration stack and store the result as the final
375:         # value in the config ivar.
376:         config_loader.load
377:       end
378: 
379:       # For the global configuration, we only need to load the configuration
380:       # in a single pass, since nothing is conditional on the configuration.
381:       global = inner_load.call
382: 
383:       # For each virtual machine represented by this environment, we have
384:       # to load the configuration in two-passes. We do this because the
385:       # first pass is used to determine the box for the VM. The second pass
386:       # is used to also load the box Vagrantfile.
387:       defined_vm_keys = global.vm.defined_vm_keys.dup
388:       defined_vms     = global.vm.defined_vms.dup
389: 
390:       # If this isn't a multi-VM environment, then setup the default VM
391:       # to simply be our configuration.
392:       if defined_vm_keys.empty?
393:         defined_vm_keys << DEFAULT_VM
394:         defined_vms[DEFAULT_VM] = Config::VMConfig::SubVM.new
395:       end
396: 
397:       vm_configs = defined_vm_keys.map do |vm_name|
398:         @logger.debug("Loading configuration for VM: #{vm_name}")
399: 
400:         subvm = defined_vms[vm_name]
401: 
402:         # First pass, first run.
403:         config = inner_load[subvm]
404: 
405:         # Second pass, with the box
406:         config = inner_load[subvm, boxes.find(config.vm.box)]
407:         config.vm.name = vm_name
408: 
409:         # Return the final configuration for this VM
410:         config
411:       end
412: 
413:       # Finally, we have our configuration. Set it and forget it.
414:       @config = Config::Container.new(global, vm_configs)

copy_insecure_private_key()

This method copies the private key into the home directory if it doesn’t already exist.

This must be done because `ssh` requires that the key is chmod 0600, but if Vagrant is installed as a separate user, then the effective uid won’t be able to read the key. So the key is copied to the home directory and chmod 0600.

Source: show 

     # File lib/vagrant/environment.rb, line 464

468:                      @default_private_key_path)
469:       end

Class Public methods
new(opts=nil)

Initializes a new environment with the given options. The options is a hash where the main available key is `cwd`, which defines where the environment represents. There are other options available but they shouldn’t be used in general. If `cwd` is nil, then it defaults to the `Dir.pwd` (which is the cwd of the executing process).

close_on_exec(file)

By default, a child process will inherit open file descriptors from the parent process. This helper provides a cross-platform way of making sure that doesn’t happen for the given file/io.

Source: show 

     # File lib/childprocess.rb, line 141
141:     def close_on_exec(file)
142:       if file.respond_to?(:close_on_exec=)
143:         file.close_on_exec = true
144:       elsif file.respond_to?(:fcntl) && defined?(Fcntl::FD_CLOEXEC)
145:         file.fcntl Fcntl::F_SETFD, Fcntl::FD_CLOEXEC
146:       elsif windows?
147:         Windows::Lib.dont_inherit file
148:       else
149:         raise Error, "not sure how to set close-on-exec for #{file.inspect} on #{platform_name.inspect}"
150:       end
151:     end

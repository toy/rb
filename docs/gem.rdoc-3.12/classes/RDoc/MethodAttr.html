Attempts to sanitize the content passed by the ruby parser: remove outer parentheses, etc.

Source: show 

     # File lib/rdoc/method_attr.rb, line 203
203:   def block_params=(value)
204:     # 'yield.to_s' or 'assert yield, msg'
205:     return @block_params = '' if value =~ /^[\.,]/
206: 
207:     # remove trailing 'if/unless ...'
208:     return @block_params = '' if value =~ /^(if|unless)\s/
209: 
210:     value = $1.strip if value =~ /^(.+)\s(if|unless)\s/
211: 
212:     # outer parentheses
213:     value = $1 if value =~ /^\s*\((.*)\)\s*$/
214:     value = value.strip
215: 
216:     # proc/lambda
217:     return @block_params = $1 if value =~ /^(proc|lambda)(\s*\{|\sdo)/
218: 
219:     # surrounding +...+ or [...]
220:     value = $1.strip if value =~ /^\+(.*)\+$/
221:     value = $1.strip if value =~ /^\[(.*)\]$/
222: 
223:     return @block_params = '' if value.empty?
224: 
225:     # global variable
226:     return @block_params = 'str' if value =~ /^\$[&0-9]$/
227: 
228:     # wipe out array/hash indices
229:     value.gsub!(/(\w)\[[^\[]+\]/, '\1')
230: 
231:     # remove @ from class/instance variables

243:           downcase
244:       else

246:       end
247:     end

documented?()

A method/attribute is documented if any of the following is true:

  • it was marked with :nodoc:;
  • it has a comment;
  • it is an alias for a documented method;
  • it has a see method that is documented.

output_name(context)

Name for output to HTML. For class methods the full name with a “.” is used like SomeClass.method_name. For instance methods the class name is used if context does not match the parent.

This is to help prevent people from using :to call class methods.

see()

A method/attribute to look at, in particular if this method/attribute has no documentation.

It can be a method/attribute of the superclass or of an included module, including the Kernel module, which is always appended to the included modules.

Returns nil if there is no such method/attribute. The is_alias_for method/attribute, if any, is not included.

Templates may generate a “see also …” if this method/attribute has documentation, and “see …” if it does not.

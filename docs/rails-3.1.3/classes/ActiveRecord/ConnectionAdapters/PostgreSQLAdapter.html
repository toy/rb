Class ActiveRecord::ConnectionAdapters::PostgreSQLAdapter < AbstractAdapter

  • activerecord-3.1.3/lib/active_record/connection_adapters/postgresql_adapter.rb

The PostgreSQL adapter works both with the native C (ruby.scripting.ca/postgres/) and the pure Ruby (available both as gem and from rubyforge.org/frs/?group_id=234&release_id=1944) drivers.

Options:

  • :host - Defaults to “localhost”.
  • :port - Defaults to 5432.
  • :username - Defaults to nothing.
  • :password - Defaults to nothing.
  • :database - The name of the database. No default, must be provided.
  • :schema_search_path - An optional schema search path for the connection given as a string of comma-separated schema names. This is backward-compatible with the :schema_order option.
  • :encoding - An optional client encoding that is used in a SET client_encoding TO <encoding> call on the connection.
  • :min_messages - An optional client min messages that is used in a SET client_min_messages TO <min_messages> call on the connection.

      # File activerecord-3.1.3/lib/active_record/connection_adapters/postgresql_adapter.rb, line 1033
1033:         def exec_cache(sql, binds)
1034:           begin
1035:             stmt_key = prepare_statement sql
1036: 
1037:             # Clear the queue
1038:             @connection.get_last_result
1039:             @connection.send_query_prepared(stmt_key, binds.map { |col, val|
1040:               type_cast(val, col)
1041:             })

1043:             @connection.get_last_result
1044:           rescue PGError => e
1045:             # Get the PG code for the failure.  Annoyingly, the code for
1046:             # prepared statements whose return value may have changed is
1047:             # FEATURE_NOT_SUPPORTED.  Check here for more details:
1048:             # http://git.postgresql.org/gitweb/?p=postgresql.git;a=blob;f=src/backend/utils/cache/plancache.c#l573
1049:             code = e.result.result_error_field(PGresult::PG_DIAG_SQLSTATE)
1050:             if FEATURE_NOT_SUPPORTED == code
1051:               @statements.delete sql_key(sql)

1053:             else

1055:             end
